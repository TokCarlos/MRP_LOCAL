RELATORIO DA ETAPA
Versao: v0.1.053-backend-minimo-api-produtos

OBJETIVO
- Criar primeiro backend local controlado com FastAPI e integrar a aba Produtos ao endpoint de API.

PROBLEMA
- A aba Produtos dependia de leitura direta do seed sem backend local minimo.

CAUSA
- O projeto estava em fase preparatoria de backend, sem servidor API em execucao.

SOLUCAO
- Criar backend minimo em `01-mrp/backend/app` com rotas:
  - `GET /health`
  - `GET /api/status`
  - `GET /api/produtos`
- Criar scripts operacionais backend em `03-vs/scripts/servicos`.
- Ajustar frontend Produtos para consumir API primeiro com fallback explicito.

ESCOPO
- sem PostgreSQL;
- sem SQLite;
- sem CRUD completo;
- sem autenticacao nova;
- sem alteracao de seed/imagens.
